Detailed Description

template<class T, std::size_t NBlock, template< typename > class BuffInitAlloc = std::allocator>
requires std::is_same_v<T, std::remove_reference_t<T>>
class tsds::PoolAlloc< T, NBlock, BuffInitAlloc >

A fix-sized pool allocator.

Template Parameters
TThe type to alloc.
NBlockThe maximum number of T that can be held.
BuffInitAllocThe allocator used to allocate the buffer. Only used twice, one by calling allocate(1) and one by calling deallocate(buf_ptr, 1). Must be copy and move constructible/assignable as per Allocator named requirement. Doesn't need to satisfy any other condition in the named requirement.
Important
This class does NOT satisfies the named requirement Allocator.
At the moment of writing, shared_ptr is not constexpr. So, as much as we hate it, this allocator is not constexpr.
Note
In constexpr context, all allocations fall back to ::operator new
Todo
Implement construct and destruct.

Manages a buffer allocated by BuffInitAlloc.

As for why that's the case: this allocator only allows you to get one chunk of memory of size sizeof(T) each time. The named requirement for Allocator requires allocate to be able to take in an extra parameter n, and return a pointer to T[n]. This allocator simply doesn't do that.

If you're sure a container will only request, one at a time, a block of the same size, you can use it like, say, std::allocator.

Parameters
t_p_objThe memory. Must be allocated by the this allocator, or any allocator that is equal to this allocator instance. Can be nullptr, at which point, the function does nothing. After the deallocation, the pointer becomes invalidated and is set to nullptr.
Invariant
t_p_obj must have been an allocation by this allocator or an allocator alloc such that:
*this == alloc;
Important
Please only let exactly one thread deallocate. Fortunately, this is the behavior that smart pointers support.
At the moment of writing, shared_ptr is not constexpr. So, as much as we hate it, this allocator is not constexpr.

Behavior is undefined if the t_p_obj passed in doesn't come from calling allocate on an instance of PoolAlloc that is not equal to this instance. Takes one parameter of type std::size_t, but is ignored. Only there to meet Allocator named requirement somewhat.
