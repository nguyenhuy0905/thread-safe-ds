Detailed Description

template<class T, std::size_t NBlock, template< typename > class BuffInitAlloc = std::allocator>
requires std::is_same_v<T, std::remove_reference_t<T>>
class tsds::PoolAlloc< T, NBlock, BuffInitAlloc >

A fix-sized pool allocator.

Template Parameters
TThe type to alloc.
NBlockThe maximum number of T that can be held.
BuffInitAllocThe allocator used to allocate the buffer. Only used once.
Important
This class does NOT satisfies the named requirement Allocator.
At the moment of writing, shared_ptr is not constexpr. So, as much as we hate it, this allocator is not constexpr.
This class holds a shared_ptr (whose constructor is, currently, the default allocator) to the allocation buffer.
Note
In constexpr context, all allocations fall back to ::operator new

Manages a buffer allocated by BuffInitAlloc.

As for why that's the case: this allocator only allows you to get one chunk of memory of size sizeof(T) each time. The named requirement for Allocator requires allocate to be able to take in an extra parameter n, and return a pointer to T[n]. This allocator simply doesn't do that.

Parameters

Invariant

*this == alloc;
Important
Please only let exactly one thread deallocate. Fortunately, this is the behavior that smart pointers support.
